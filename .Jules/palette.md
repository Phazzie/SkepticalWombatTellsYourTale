## 2024-03-20 - Explicit Focus States and Contextual ARIA labels required for Custom Grid Elements
**Learning:** This design system heavily utilizes custom `<button>` elements that visually resemble cards or list items, and repeating inline action buttons without distinct labels. In these environments, native focus styles are often suppressed or insufficient. Also, inline elements need contextual names. Screen reader users get no context if 5 buttons on screen simply say "Resolved".
**Action:** Always proactively apply `focus-visible` ring classes (e.g., `focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-*`) to any interactive element masquerading as a card or custom layout block to ensure keyboard navigation visibility. For repeating lists of items with actions, always dynamically inject the target's name into the `aria-label` (e.g., `aria-label={"Resolve: " + item.name}`).
